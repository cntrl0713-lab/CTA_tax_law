# AI 채점 플랫폼 세법 문제 ID 설정 규칙

세법 문항 데이터를 데이터베이스(Supabase)에 새로 등록하거나 마이그레이션할 때, 아래의 ID 매핑 규칙을 철저하게 준수해야 합니다.

## ID 설정 공식

### 1. 문제 ID (Problem ID)

- 고유한 정수형 값입니다 (예: `47`, `48`).

### 2. 소문항 ID (Subquestion ID)

- **공식**: `문제 ID * 10 + 소문항 번호(n)`
- **예시**:
  - 문제 ID가 `47`이고, 소문항 번호가 `1`인 경우: `47 * 10 + 1 = 471`
  - 문제 ID가 `48`이고, 소문항 번호가 `2`인 경우: `48 * 10 + 2 = 482`

### 3. 루브릭 ID (Rubric ID)

- **공식**: `소문항 ID * 10 + 루브릭 표시 순서(n)`
- **예시**:
  - 소문항 ID가 `471`이고, 첫 번째 루브릭(display_order = 1)인 경우: `471 * 10 + 1 = 4711`
  - 소문항 ID가 `482`이고, 네 번째 루브릭(display_order = 4)인 경우: `482 * 10 + 4 = 4824`

---

## 데이터 적재 시 주의사항 (개발자 가이드)

- **순차적 일련번호 사용 금지**: 소문항과 루브릭을 등록할 때 단순 `Auto Increment` 대용 성격의 일련번호(예: 462 등 임의의 고유번호)를 주입해서는 안 됩니다. 반드시 상위 테이블의 ID에 곱셈 공식을 적용하여 ID를 도출해야 합니다.
- **데이터 정합성 보장**: 외부/별도 문제 업로드 애플리케이션을 통해 데이터 적재 작업을 수행할 때, 각 소문항과 루브릭의 ID가 본 규칙을 위배하지 않도록 연산 로직을 철저히 구성해야 합니다.
